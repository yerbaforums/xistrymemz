OVERVIEW

Created a simplified, community-focused fork of the xistrymemz project. The original xistrymemz had 30+ database models with complex features including:

  • Full marketplace, plans, schools, escrow, crypto wallets
  • Forums, events, groups, messaging
  • User profiles with shop/school capabilities

Earthlite simplifies this to core community features while laying the foundation for decentralized federation.

DECENTRALIZATION OPTIONS DISCUSSED

1. ActivityPub (Fedify) - Recommended

  • W3C standard, federates with Mastodon, Pixelfed, Lemmy
  • Fedify library: @fedify/fedify, @fedify/next
  • Adds WebFinger, Actor endpoints, Inbox/Outbox
  • Requires Node.js 22+

2. IPFS Storage Layer

  • Store media/content on IPFS
  • Use CIDs instead of full URLs
  • Content persists even if server goes down
  • Can combine with ActivityPub

3. Hybrid Approach (Recommended)

  • Fedify for federation protocol
  • IPFS for media storage
  • Local SQLite for user accounts/relationships
